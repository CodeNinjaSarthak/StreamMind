## Configuration

- Config: `backend/alembic.ini`
- Env: `backend/alembic/env.py`
- Migrations dir: `backend/alembic/versions/`

## Common Commands

```bash
# Create a new migration (auto-detect model changes)
cd backend
alembic revision --autogenerate -m "description of change"

# Apply all pending migrations
alembic upgrade head

# Rollback one migration
alembic downgrade -1

# Show current revision
alembic current

# Show migration history
alembic history --verbose
```

## Running Against Docker Postgres

In development (Docker not always available — see note below):

```bash
docker-compose up -d postgres
cd backend
DATABASE_URL=postgresql://user:password@localhost:5432/streammind_dev alembic upgrade head
```

DB credentials (docker-compose): user=`user`, password=`password`, db=`streammind`

> **Note:** Docker was not available in the original dev environment. Some migrations
> were written manually rather than generated by Alembic.

## Manual Migration Note

When autogenerate is not possible (e.g., pgvector column types), write the migration
manually in `backend/alembic/versions/`. Follow the existing migration file structure.

## Conflict Resolution

If two branches create conflicting migration files (same `down_revision`):
1. Identify the conflict: `alembic heads` shows multiple heads
2. Create a merge migration: `alembic merge heads -m "merge migrations"`
3. Apply: `alembic upgrade head`

## pgvector Setup

The `pgvector` extension must be enabled in the database before migrations run:

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

This is typically done in the initial migration or via `docker-compose` init scripts.
